# CID Core Scope

**Date:** 2026-05-21
**Phase:** PRODUCT.2

## CID Core

CID Core se centra en preproduccion cinematografica y en los entregables que ayudan a convertir un guion en una base clara para decidir, presentar y producir.

### Entra en CID Core

- guion
- analisis cinematografico
- secuencias
- storyboard
- visual bible
- pitch / dossier
- presupuesto
- documentos de produccion
- previs / video basico

### Queda fuera de CID Core

- doblaje
- arreglo o restauracion de sonido
- restauracion de imagenes

## Productos separados propuestos

### DubbingTake Studio AI

- foco: sesiones, takes, QC de doblaje, sincronias, trazabilidad vocal
- encaje: producto de audio/voz separado del flujo base de preproduccion

### Sound Post AI

- foco: limpieza, spotting, stems, materiales de audio para post y delivery
- encaje: producto de sound post separado del recorrido CID Core

### Restoration Lab AI

- foco: repair, cleanup, conform, enhancement y restauracion de imagen
- encaje: laboratorio tecnico independiente, no parte del catalogo cliente de CID Core

## Justificacion

1. El valor comercial mas claro de CID hoy esta en preproduccion y desarrollo.
2. Doblaje, sound post y restoration introducen dependencias tecnicas, legales y operativas distintas.
3. Mantener esos bloques dentro del relato principal de CID mezcla jobs creativos de preproduccion con operaciones especializadas de post.
4. El reenfoque reduce confusion comercial, simplifica navegacion y alinea el producto con el estado real del sistema.
5. No se pierde infraestructura interna: rutas, endpoints y workflows permanecen disponibles para uso interno o evolucion futura.

## Cambios de producto recomendados

1. Ocultar del catalogo CID Core los modulos no-core visibles al cliente.
2. Mantener las rutas vivas, pero degradarlas a "laboratorio interno / futuro producto" cuando se accedan directamente.
3. Reposicionar el copy publico de CID en torno a preproduccion cinematografica.
4. Mantener editorial / previs basica dentro del core mientras no derive a sound post o restoration.

## Roadmap recomendado

### Bloque 1

- consolidar CID Core en frontend y catalogo modular
- limpiar narrativas comerciales que aun prometen un sistema audiovisual total
- reforzar storyboard, visual bible, pitch y presupuesto

### Bloque 2

- crear start path seguro para restoration sin auto-install
- reparar 8191 en entorno aislado
- definir readiness tecnica de Restoration Lab AI

### Bloque 3

- preparar ola critica de modelos faltantes para storyboard/image y video/cine
- medir impacto real en workflows CID core

### Bloque 4

- decidir si DubbingTake Studio AI y Sound Post AI salen como productos separados visibles o siguen internos
- evaluar marca, pricing y onboarding propios para esas lineas

## Recomendacion final

- **CID Core: GO** como producto principal de preproduccion.
- **DubbingTake Studio AI: separado**.
- **Sound Post AI: separado**.
- **Restoration Lab AI: separado y en modo laboratorio**.
