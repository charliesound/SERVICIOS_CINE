# AILink Sync Dialogue — Demo Video Subtitles v1

## 1. Objetivo

Este pack prepara subtítulos y texto de locución para el vídeo demo de AILink Sync Dialogue.

No renderiza vídeo, no crea MP4 y no publica nada. Solo deja preparados textos reutilizables para grabación, edición y revisión.

## 2. Archivos

- `ailink_sync_dialogue_demo_video_subtitles_es_v1.srt`: subtítulos en español para vídeo aproximado de 90–95 segundos.
- `ailink_sync_dialogue_demo_video_voiceover_es_v1.txt`: texto limpio para locución.
- `ailink_sync_dialogue_demo_video_subtitles_readme_v1.md`: este documento.

## 3. Uso previsto

Estos textos pueden usarse para:

- Grabar voz en off.
- Montar una demo de 60–90 segundos.
- Generar subtítulos en un editor de vídeo.
- Revisar el mensaje antes de publicar.
- Adaptar una versión corta para LinkedIn o Facebook.

## 4. Mensaje principal

**Prepara material antes de montaje.**

La demo explica que AILink Sync Dialogue:

- Escanea carpetas locales.
- Detecta vídeo y audio.
- Lee metadata cuando está disponible.
- Sugiere posibles matches.
- Genera `report.html`, `media_files.csv`, `match_suggestions.csv` y `scan_result.json`.
- Trabaja localmente en la versión actual.
- No sube material audiovisual a la nube.
- Está en beta privada.

## 5. Limitaciones comunicadas

El texto deja claro que la versión actual todavía no incluye:

- Waveform sync.
- Transcripción.
- Detección de claqueta visual.
- Instalador final.
- Integración directa con DaVinci Resolve, Avid Media Composer o Adobe Premiere Pro.

## 6. Checklist antes de grabar

Antes de grabar o editar:

- Revisar pronunciación de AILink Sync Dialogue.
- Mantener ritmo natural.
- No correr en los bloques largos.
- Dejar pausas entre ideas.
- No añadir promesas que no estén en el guion.
- No decir que sustituye al montador.
- No decir que sincroniza todo automáticamente.
- No pedir envío de material audiovisual.
- Usar solo assets auditados.
- No mostrar datos reales.
- No mostrar rutas personales.

## 7. Non-goals de esta fase

Esta fase no debe:

- Crear vídeo.
- Crear audio.
- Crear MP4.
- Crear subtítulos finales incrustados.
- Crear assets nuevos.
- Tocar frontend.
- Tocar backend CID.
- Tocar Supabase.
- Tocar CRM.
- Tocar `.env`.
- Añadir dependencias.
- Publicar nada.

## 8. Criterios de aceptación

La fase se considera válida si:

- Existe un `.srt` en español.
- Existe texto de locución.
- Los subtítulos tienen numeración secuencial.
- Los timestamps avanzan en orden.
- Se mencionan outputs reales.
- Se comunica local-first.
- Se comunica beta privada.
- Se comunican limitaciones honestas.
- No se prometen funciones inexistentes.
