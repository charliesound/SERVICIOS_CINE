Define the exact contract for a visible internal demo report based on the current local scanner baseline.

This phase is docs/test-only.

This phase does not execute the scanner.

This phase does not generate the report file.

This phase does not modify runtime code.

## Report Identity

The future report is an internal demo report.

Canonical report title:

`CID Local Media Agent - Internal Demo Visible Report`

Canonical report audience:

`producer_product_post_internal_review`

Canonical report status:

`internal_demo_only`

Canonical privacy mode:

`local_only`

## Input Scope Contract

The report must describe only the approved synthetic controlled demo scenario.

The report must not describe real client media.

The report must not describe public demo material.

The report must not describe production customer footage.

The report must not claim SaaS upload or cloud processing.

## Accepted Output Families

The report may use only the accepted current scanner output families:

- `00_project/`
- `01_media_catalog/`
- `99_logs/`

The report must not require these future output families to exist:

- `02_audio_sync/`
- `03_transcription/`
- `04_subtitles/`
- `05_reports/`
- `06_exports/`

Those future output families must be labelled as roadmap modules.

## Required Report Sections

The report must contain exactly these top-level visible sections in this order:

1. `Executive Summary`
2. `Local-Only Privacy Confirmation`
3. `Controlled Demo Input Summary`
4. `Scanner Result Summary`
5. `Accepted Media`
6. `Rejected Non-Media`
7. `Human Review Required`
8. `Warnings`
9. `Created Output Artifacts`
10. `Roadmap Modules Not Yet Generated`
11. `Producer Interpretation`
12. `Next Technical Actions`

## Required Executive Summary Content

The executive summary must state:

- the scanner completed with warnings
- the execution stayed local-only
- valid media candidates were detected
- non-media files were rejected
- at least one item requires human review
- no sync, transcription, subtitles, or export module was executed

## Required Data Contract

The report must expose these fields when generated from the current controlled demo evidence:

- `status = completed_with_warnings`
- `privacy_mode = local_only`
- `candidate_media_count = 5`
- `human_review_required_count = 1`
- `warnings_count = 1`
- `warning_message = unknown synthetic placeholder`
- `ffprobe_preflight = skipped`
- `original_media_left_client_system = false`

## Required Accepted Media Summary

The accepted media section must include:

- `.mov = 1`
- `.mp4 = 2`
- `.wav = 1`

The section must explain that accepted media are scanner candidates, not edited deliverables.

## Required Rejected Non-Media Summary

The rejected non-media section must include:

- `.exe = 1`
- `.txt = 2`

The section must explain that non-media rejection is expected and protects the media catalog.

## Required Human Review Summary

The human review section must state:

- one synthetic unknown placeholder requires human review
- the warning-path exit code is expected
- ambiguous material is not silently accepted

## Required Created Artifacts Summary

The created artifacts section must mention the current output families:

- project-level scanner artifacts under `00_project/`
- media catalog artifacts under `01_media_catalog/`
- privacy and processing logs under `99_logs/`

The report must avoid over-claiming exact runtime filenames unless they are present in the inspected scanner output.

## Required Roadmap Disclosure

The roadmap section must explicitly state that these modules are not yet generated by the current scanner baseline:

- audio synchronization
- transcription
- subtitle generation
- translation
- report generation runtime
- DaVinci Resolve export
- Avid export

The roadmap section must map these modules to future output families:

- `02_audio_sync/`
- `03_transcription/`
- `04_subtitles/`
- `05_reports/`
- `06_exports/`

## Producer Interpretation Contract

The producer interpretation must use plain production language.

It must explain that the current scanner baseline is useful for organizing and validating local media intake before later post-production intelligence modules.

It must not imply that the current baseline can already synchronize audio, transcribe dialogue, generate subtitles, or export Resolve timelines.

## Privacy Contract

The report must not leak:

- local user names
- machine names
- Windows paths
- WSL paths outside approved synthetic demo context
- repository paths
- real client material
- private project titles
- private filenames from real shoots

Forbidden tokens include:

- `/mnt/`
- Windows drive paths
- UNC paths
- `DESKTOP-`
- `harliesound`
- `SERVICIOS_CINE`

## Non-Goals

This contract does not authorize:

- runtime report generation
- scanner implementation changes
- real media scanning
- public demo use
- client-facing demo use
- ffprobe execution
- ffmpeg execution
- SaaS upload
- database writes
- network calls
- Docker changes
- Alembic changes
- frontend/backend SaaS changes
- Stripe, AI Jobs, credits, or ledger changes

## Acceptance Criteria

The visible report contract is accepted when:

- all required sections are specified
- current scanner output families are the only accepted source families
- future modules are labelled as roadmap
- privacy restrictions are explicit
- producer-facing interpretation is required
- runtime implementation remains out of scope
