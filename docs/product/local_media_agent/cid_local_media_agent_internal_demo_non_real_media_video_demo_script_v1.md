## Demo Script

### 1. Opening

Explain that this is an internal controlled demo of the Local Media Agent direction.

Do not claim that the product is ready for clients.

Do not claim that the product can already process real footage.

### 2. Problem

Explain the production problem:

Film and documentary projects often receive large folders of camera, audio, proxy, reference and delivery material. The production team needs a local tool that can organize, inspect and later prepare media intelligence without uploading the client files.

### 3. Product Direction

Explain the intended direction:

CID Local Media Agent is designed as a local-only companion for production and post-production teams. The long-term goal is to scan local project folders, identify media structure, prepare reports, and later support synchronization, transcription, subtitles and editorial handoff.

### 4. Current Safe Status

- repository stable state
- closed controlled execution chain
- closed checklist summary
- closed QA gates
- local-only boundary
- controlled-fixture-only boundary
- blocked capabilities

### 5. What Can Be Shown

### 6. What Must Not Be Shown

### 7. Future Path

### 8. Closing
