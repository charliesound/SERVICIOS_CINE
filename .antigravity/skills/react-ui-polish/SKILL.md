# React UI Polish

Use this skill when polishing React/Tailwind interfaces for clarity, responsiveness, and interaction quality.

## Layout Rules

- Build mobile-first and validate desktop spacing explicitly.
- Use consistent spacing rhythm and avoid cramped card interiors.
- Favor simple composition over nested ornamental wrappers.

## Interaction Rules

- Use a small number of meaningful hover and reveal states.
- Keep CTA states obvious and accessible.
- Avoid excessive animation or decorative motion that distracts from hierarchy.

## Component Rules

- Prefer reusable section, card, and hero primitives over one-off markup.
- Keep prop shapes small and readable.
- Make empty states, labels, and support text useful and intentional.

## Visual Rules

- Preserve strong contrast in dark themes.
- Use restrained accent color and subtle gradients.
- Avoid default-looking UI with weak hierarchy or ambiguous calls to action.
