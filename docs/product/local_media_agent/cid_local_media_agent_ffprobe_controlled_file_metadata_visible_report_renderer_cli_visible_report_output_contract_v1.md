# CID Local Media Agent - FFprobe Controlled File Metadata Visible Report Renderer CLI Visible Report Output Contract v1

## Phase

`CID.LOCAL_MEDIA_AGENT.FFPROBE.CONTROLLED.FILE.METADATA.PREFLIGHT.VISIBLE.REPORT.RENDERER.CLI.VISIBLE.REPORT.OUTPUT.CONTRACT.V1`

## Objective

Define the visible report output contract generated by the local-only CLI from already validated synthetic controlled JSON.

This phase is document-contract only.

There is no runtime implementation.

There are no CLI changes.

There are no renderer changes.

This phase does not execute the CLI.

This phase does not execute the renderer.

This phase does not execute real ffprobe.

This phase does not execute ffmpeg.

## Previous Closed Phase

`CID.LOCAL_MEDIA_AGENT.FFPROBE.CONTROLLED.FILE.METADATA.PREFLIGHT.VISIBLE.REPORT.RENDERER.CLI.SMOKE.CONTROLLED.JSON.FIXTURE.QA.GATE.V1`

## Authorized Chain

The only authorized chain remains:

controlled synthetic JSON fixture
-> existing local-only CLI
-> existing pure renderer
-> safe visible report text

## Allowed Visible Output

The visible report may show only safe controlled technical information derived from synthetic controlled JSON.

Allowed visible output includes:

- controlled report title
- phase identifier
- controlled input policy
- redacted input marker or controlled filename
- preflight result from controlled JSON
- safe format summary
- safe stream summary
- video stream summary from synthetic metadata
- audio stream summary from synthetic metadata
- unknown stream summary when present
- safety boundary summary
- blocked operations summary
- human review required note
- next safe phase
- safety flags when present, especially as `false`

## Forbidden Visible Output

The visible report must never show:

- full paths
- arbitrary folders
- real project names
- real shoot names
- real rodaje names
- real media material
- local users
- home paths
- absolute paths
- environment values
- secrets
- DB/SaaS data
- network data
- audiovisual content
- private material
- client names
- production names

## Redaction/Privacy

The visible report must use already redacted identifiers or controlled filenames only.

If an unsafe path, local user, environment value, absolute path, private identifier, client name, project name, production name, or real media marker is present upstream, the future visible output must omit it or replace it with a safe redacted marker.

The visible report must preserve redaction boundaries from the controlled JSON fixture, existing local-only CLI, and existing pure renderer.

## Format Contract

The visible report output is authorized only as human-readable plain text or Markdown-safe text.

The output may be printed to stdout by the existing local-only CLI or written as a controlled `.txt` or `.md` report when that behavior is already authorized by the CLI phase.

The output must be deterministic for the same controlled JSON fixture.

The output must not include timestamps, hostnames, machine names, usernames, absolute paths, environment values, network identifiers, database identifiers, or SaaS identifiers.

## Blocked Boundaries

This contract does not authorize:

- real media
- scanner execution
- real ffprobe execution
- ffmpeg
- subprocess/process execution
- audio extraction
- sync
- transcription
- subtitles
- timeline export
- network
- SaaS/DB
- installer
- public demo
- client demo
- sales demo
- production use
