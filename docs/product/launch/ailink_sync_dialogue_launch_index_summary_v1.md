# AILink Sync Dialogue — Launch Index Summary v1

## 1. Objetivo

Este documento resume el pack comercial actual de AILink Sync Dialogue y sirve como índice operativo para saber qué está listo, qué se puede enseñar, qué no debe prometerse y qué falta antes de una publicación real.

Esta fase no implementa landing real, formulario, CRM, Supabase, vídeo, tracking, cookies, frontend, backend, CID SaaS, Docker, runtime ni configuración.

## 2. Estado actual estable

AILink Sync Dialogue dispone actualmente de:

- Prototipo local de scanner/matching/export.
- Demo técnica con metadata controlada.
- Reporte HTML.

- CSV de sugerencias de match.
- JSON de resultado.
- Landing HTML estática exportable.
- Assets visuales controlados.
- Copy comercial de landing.
- Pack social para LinkedIn/Facebook.
- Textos legales base.
- Spec de integración legal del formulario beta.
- Guion de vídeo demo.
- Subtítulos SRT.
- Texto de locución.
- Runbook de montaje de vídeo.
- Spec de production pack de vídeo.
- Runbook manual de leads beta.

## 4. Qué se puede enseñar ya

Se puede enseñar internamente o en demo controlada:

- La landing estática exportable.
- El reporte HTML de la demo controlada.
- Los CSV de ejemplo.
- Los assets visuales aprobados.
- El mensaje local-first.
- El flujo manual de leads.
- El guion del vídeo demo.
- La explicación de beta privada.
- La propuesta comercial a escuelas, productoras y equipos de postproducción.

## 5. Qué no se debe enseñar como producto terminado

No presentar como terminado:

- SaaS público.
- Formulario funcional.
- CRM.
- Integración Supabase activa.
- Instalador.
- Integración directa con DaVinci Resolve, Avid o Premiere.
- Waveform sync.
- Transcripción.
- Reconocimiento visual de claqueta.
- Sincronización automática completa.
- Edición automática.
- Producto cerrado para producción real.

## 6. Mensaje comercial aprobado

Mensaje corto:

AILink Sync Dialogue prepara vídeo, audio y metadata antes de entrar en montaje.

Mensaje ampliado:

AILink Sync Dialogue es una herramienta local-first para analizar carpetas de material audiovisual, detectar vídeo y audio, extraer metadata disponible, sugerir relaciones entre archivos y generar salidas revisables para equipos de montaje y postproducción.

Mensaje de beta:

La beta privada se orienta a escuelas, productoras y equipos de postproducción que quieran probar la herramienta con casos controlados y aportar feedback concreto.

## 7. Público objetivo prioritario

Prioridad alta:

- Escuelas de cine.
- Productoras pequeñas y medianas.
- Equipos de postproducción.
- Montadores.
- DITs.
- Ayudantes de montaje.
- Coordinadores de postproducción.

Prioridad media:

- Profesionales audiovisuales independientes.
- Creadores con flujo real de vídeo/audio.
- Formadores audiovisuales.
- Productores con necesidad de ordenar material.

No prioritario ahora:

- Público general.
- Usuarios que buscan solo IA generativa visual.
- Clientes que necesitan SaaS completo.
- Clientes que exigen integración NLE inmediata.
- Clientes que quieren subir material sensible sin marco legal preparado.

## 8. Orden recomendado de uso comercial

Orden recomendado:

1. Revisar landing estática.
2. Revisar assets.
3. Revisar reporte HTML de demo.
4. Revisar social launch pack.
5. Revisar textos legales base.
6. Revisar flujo manual de leads.
7. Preparar vídeo demo cuando se autorice.
8. Contactar leads de alta calidad manualmente.
9. Registrar feedback.
10. Decidir si se implementa formulario real o CRM.

## 9. Checklist antes de enseñar a terceros

Antes de enseñar:

- Confirmar que se usa demo controlada.
- Confirmar que no aparece material real de clientes.
- Confirmar que no aparecen secretos.
- Confirmar que no se piden archivos audiovisuales.
- Confirmar que se explica beta privada.
- Confirmar que no se promete acceso inmediato.
- Confirmar que no se promete waveform sync.
- Confirmar que no se promete transcripción.
- Confirmar que no se promete claqueta visual.
- Confirmar que no se promete integración directa con editores.
- Confirmar que se separa AILink Sync Dialogue de CID.
- Confirmar que el mensaje coincide con la landing.
- Confirmar que el contacto se gestiona con el runbook de leads.

## 10. Checklist antes de publicar landing real

Antes de publicar landing real:

- Completar placeholders legales.
- Revisar textos con asesoría legal.
- Decidir dominio.
- Definir si habrá formulario o solo contacto manual.
- Si hay formulario, definir almacenamiento.
- Si hay CRM/Supabase, abrir fase específica.
- Revisar cookies/tracking.
- Revisar consentimiento comercial.
- Revisar política no_contactar.
- Revisar retención de leads.
- Revisar copies de redes.
- Revisar vídeo o decidir publicar sin vídeo.

## 11. Próximas decisiones

Decisiones pendientes:

- Publicar landing estática o esperar a formulario.
- Contacto manual o formulario beta.
- Supabase o CRM posterior.
- Vídeo demo real ahora o más adelante.
- Dominio principal.
- Precio beta inicial.
- Lista inicial de escuelas/productoras a contactar.
- Mensaje final de LinkedIn.
- Fecha de primera publicación.

## 12. Recomendación operativa

Recomendación:

No implementar CRM, Supabase ni formulario todavía si no hay leads reales.

Primero conviene revisar el pack, preparar 5 a 10 contactos de alta calidad, enseñar la demo controlada y validar si el mensaje comercial despierta interés real.

Solo después tiene sentido automatizar captación.

## 13. Relación con CID

AILink Sync Dialogue es una herramienta independiente.

CID sigue siendo el SaaS integral de AILinkCinema.

Reglas:

- No vender AILink Sync Dialogue como CID.
- No usar cuenta CID como requisito.
- No prometer módulos CID desde la herramienta independiente.
- Se puede mencionar CID como visión futura de AILinkCinema, pero separado de esta beta.
